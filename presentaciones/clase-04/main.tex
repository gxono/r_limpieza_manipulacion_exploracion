	\begin{frame}[fragile]
		Una vez que tenemos nuestros datos cargados en R, comienza lo más divertido: transformar esos datos crudos en información útil. R es excepcionalmente bueno para esto gracias al ecosistema \rscript{tidyverse}, que implementa una \quotation{gramática de manipulación de datos} coherente e intuitiva.
		
		En este encuentro nos centraremos en entender el flujo de trabajo moderno en R y en las operaciones que afectan a las columnas (variables) de nuestros datos: seleccionar las que nos interesan y crear nuevas a partir de las existentes.
	\end{frame}






	\begin{frame}[fragile]{El operador \quotation{pipe} }
		Posiblemente la herramienta más revolucionaria para la legibilidad del código en R. El pipe (tubería) nos permite encadenar operaciones secuencialmente, leyendo el código de izquierda a derecha y de arriba a abajo, tal como leemos un texto.
		
		\begin{figure}[h!]
			\includegraphics[width = 0.15\textwidth]{img/magrittr.png}
		\end{figure}
	\end{frame}
	
	
	
	
	
	
	\begin{frame}[fragile]
		En términos prácticos, el \quotation{pipe} toma el resultado de la expresión de la izquierda y lo pasa como \quotation{primer argumento} de la función de la derecha. 
		
		Es decir, en lugar de escribir \rscript{f(g(h(x)))}, podemos escribir \rscript{x |> h() |> g() |> f()}. 
		
		Esto no cambia lo que el programa hace, pero sí cambia radicalmente cómo se lee.
		\begin{itemize}
			\item \rscript{x |> f} es equivalente a \rscript{f(x)}
			\item \rscript{x |> f(y)} es equivalente a \rscript{f(x, y)}
			\item \rscript{x |> f |> g |> h} es equivalente a \rscript{h(g(f(x)))}
		\end{itemize}
	\end{frame}




	\begin{frame}[fragile]
		¿Por qué es necesario? Porque en análisis de datos es común aplicar muchas transformaciones seguidas: filtrar, ordenar, crear variables, agrupar, resumir, etc. Sin *pipe*, el código suele terminar en llamadas anidadas (hay que leer \quotation{desde adentro hacia afuera} ) o en una secuencia de asignaciones intermedias (\rscript{x1}, \rscript{x2}, \rscript{x3}), que fragmenta el flujo lógico y dificulta seguir el proceso.
	\end{frame}
	
	
	
	
	\begin{frame}[fragile]
		Imagina este proceso en la vida real: \quotation{Tamizar harina, templar el agua, hidratar, mezclar con sal y levadura, amasar, dejar leudar, formar bollos y hornear} . Si quisieramos escribir este proceso como una sucesion de funciones en R deberiamos hacer algo como
		
\begin{lstlisting}[style=RScript]
panes <- `:fn:hornear`(`:fn:formar`(`:fn:leudar`(`:fn:amasar`(`:fn:mezclar`(`:fn:hidratar`(`:fn:tamizar`(harina), agua = `:fn:templar`(agua, 28)), sal = 10, levadura = 3), minutos = 12), minutos = 60, temperatura = 26), forma = "bollos"), minutos = 25, temperatura = 220)
\end{lstlisting}

u \quotation{organizando mejor el código} 
\begin{lstlisting}[style=RScript]

				`:fn:amasar`(
					`:fn:mezclar`(
						`:fn:hidratar`(
							`:fn:tamizar`(harina), agua = `:fn:templar`(agua, 28)

		),

	)
\end{lstlisting}
	\end{frame}

\begin{frame}[fragile]
	El pipe soluciona eso al permitir escribir el código como un pipeline: una receta donde cada línea es un paso. Esto mejora:
	
	\begin{itemize}
		\item Legibilidad: el orden del código coincide con el orden mental del procedimiento.
		\item Mantenibilidad: es fácil insertar, quitar o reordenar pasos sin reescribir todo.
		\item Depuración: se pueden inspeccionar resultados intermedios agregando una línea extra (por ejemplo, para imprimir o visualizar un objeto) sin romper la estructura.
\begin{lstlisting}[style=RScript]
ventas |> view() #tibble::view()

# o también
ventas %>% view	
\end{lstlisting}
	\end{itemize}
\end{frame}






\begin{frame}[fragile]{Pipe nativo}
Introducido en la versión 4.1.0 de R (2021), es el pipe que viene integrado en el lenguaje base, sin necesidad de instalar nada.
\begin{itemize}
	\item Sintaxis: \rscript{|>}
	\item Ventaja: Más rápido y ligero. No depende de paquetes externos.
	\item Uso: Pasa el resultado de la izquierda como primer argumento de la función de la derecha.
\end{itemize}

\begin{lstlisting}[style=RScript]
# Ejemplo: Calcular la media de un vector
c(1, 2, 3, 4, 5) |> mean()

# Es exactamente lo mismo que:
mean(c(1, 2, 3, 4, 5))	
\end{lstlisting}
\end{frame}







\begin{frame}[fragile]{Pipe \texttt{magrittr}}
	Es el pipe clásico que popularizó el paquete \rscript{dplyr} y \rscript{tidyverse}. Durante años fue el único estándar.

	
	\begin{itemize}
		\item Sintaxis: \lstinline[style=RScript, basicstyle = \normalfont]`%>%`
		\item Ventaja: Más flexible (permite usar el punto \rscript{.} para referenciar el dato en lugares que no sean el primer argumento), aunque un poco más lento.
		\item \lstinline[style=RScript, basicstyle = \normalfont]`x %>% f(y, .)` es equivalente a \rscript{f(y, x)}
		\item \lstinline[style=RScript, basicstyle = \normalfont]`x %>% f(y, z = .)` es equivalente a \rscript{f(y, z = x)}
		\item Requerimiento: Necesitas cargar \rscript{library(tidyverse)} o \rscript{library(magrittr)}.
	\end{itemize}
	
	En este curso, y para el 99\% de las tareas de análisis de datos, son intercambiables. 
	
	Vas a ver mucha documentación antigua usando \lstinline[style=RScript, basicstyle = \normalfont]`%>%` y mucha nueva usando \rscript{|>}. ¡Acostúmbrate a ambos!
\end{frame}







\begin{frame}[fragile]{El paquete dplyr y los verbos para la gramática de datos}
	\rscript{dplyr} es el paquete central del tidyverse para manipulación de datos. Su filosofía se basa en funciones que hacen una sola cosa bien y tienen nombres de verbos en inglés.
	
	Te recomiendo que tengas el cheatsheet siempre a mano. Hay una copia \href{https://servicios.unl.edu.ar/aulavirtual/fce/mod/resource/view.php?id=67152&redirect=1}{\textbf{ac\'a en el aula virtual}}.
	
	Todas las funciones de \rscript{dplyr} comparten la misma estructura:
	\begin{enumerate}
		\item Primer argumento: Un data frame (o tibble).
		\item Argumentos siguientes: Qué hacer con ese data frame (nombres de columnas, fórmulas, etc.).
		\item Resultado: Siempre devuelve un nuevo data frame.
	\end{enumerate}
\end{frame}





\begin{frame}[fragile]{Los verbos m\'as importantes}
	Existen 5 verbos fundamentales que cubren la gran mayoría de tus necesidades:
	\begin{enumerate}
		\item \rscript{select()}: Selecciona columnas (variables).
		\item \rscript{filter()}: Filtra filas (observaciones).
		\item \rscript{arrange()}: Ordena filas.
		\item \rscript{mutate()}: Crea o modifica columnas.
		\item \rscript{summarise()}: Resume valores (colapsa muchas filas en una sola).
	\end{enumerate}
	
	Hoy nos centraremos en \rscript{select} y \rscript{mutate}.
\end{frame}



\begin{frame}[fragile]{El verbo \texttt{select}}
	Muchas veces nuestros datasets tienen cientos de columnas, pero solo nos interesan unas pocas. \rscript{select()} nos permite quedarnos con lo importante.
\begin{lstlisting}[style=RScript]
# Usaremos el dataset mtcars como ejemplo
# Seleccionar columnas explícitamente
mtcars |> select(`:bn:mpg`, `:bn:cyl`, `:bn:hp`)

# Seleccionar un rango de columnas (con :)
mtcars |> select(`:bn:mpg`:`:bn:drat`)

# Seleccionar todas MENOS algunas (con -)
mtcars |> select(-`:bn:am`, -`:bn:gear`, -`:bn:carb`)	
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]{Antes de continuar: helpers}
	\titleitem{Funciones de ayuda (Helpers)}
	\rscript{dplyr} ofrece funciones geniales para seleccionar columnas basándose en sus nombres, lo cual es muy útil cuando trabajamos con bases de datos grandes. Es con estos helpers que se puede ver la utilidad de tener bien asignados los nombres de las variables.

	\begin{itemize}
		\item \rscript{starts_with("prefijo")}: selecciona columnas que empiezan con un prefijo.
		\item \rscript{ends_with("sufijo")}: selecciona columnas que terminan con un sufijo.
		\item \rscript{contains("texto")}: selecciona columnas que contienen una cadena de texto.
		\item \rscript{num_range("prefijo", rango)}: selecciona columnas con un prefijo y una secuencia numérica (ej: \rscript{`x1`}, \rscript{`x2`}, \rscript{`x3`}).
		\item \rscript{all_of(`vector`)} o tambi\'en \rscript{any_of(`vector`)}: selecciona columnas cuyos nombres están en un vector de texto.
		\item \rscript{matches("expresion")}: selecciona columnas que coinciden con una expresión regular.
		\item \rscript{everything()}: selecciona todas las columnas restantes.
	\end{itemize}
\end{frame}







\begin{frame}[fragile]{}
Veamos un ejemplo más realista simulando una base de datos de ventas mensuales:
\begin{lstlisting}[style=RScript]
# Creamos un dataset de juguete para demostrar
ventas <- tibble(
	producto_id = 1:3,
	producto_nombre = c("Refresco", "Papas", "Galletas"),
	`mes_01` = c(100, 200, 150),
	`mes_02` = c(110, 210, 160),
	`mes_03` = c(120, 220, 170),
	zona_norte = c(TRUE, FALSE, TRUE),
	zona_sur = c(FALSE, TRUE, FALSE)
)

# 1. Empiezan con "producto"
ventas |> select(starts_with("producto"))

# 2. Terminan con "id"
ventas |> select(ends_with("id"))

# 3. Contienen "zona"
ventas |> select(contains("zona"))

# 4. Rango numérico (para seleccionar mes_01, mes_02, mes_03)
ventas |> select(num_range("mes_0", 1:3))

# 5. Usando un vector de nombres (tengo una entrada en el blog sobre esto.
# Comentarlo en `mutate`)
rubros <- colnames(ventas)[4:10]
ventas |> select(all_of(rubros))

# 6. Reordenar: poner las zonas al principio y luego el resto
ventas |> select(starts_with("zona"), everything())

# 7. Combinar selectores: nombre exacto y patrón (ej. solo nombre y zonas)
ventas |> select(producto_nombre, starts_with("zona"))
\end{lstlisting}
\end{frame}







\begin{frame}[fragile]{El verbo \texttt{mutate}}
	El análisis de datos casi siempre implica crear nuevas variables: pasar de pesos a dólares, calcular el IMC a partir de altura y peso, o categorizar una variable numérica. \rscript{mutate()} es la herramienta para esto.
	
	Lo importante es recordar que \rscript{mutate()} crea un nuevo data frame y agrega la nueva columna al final del data frame. Si quieres guardar el cambio, debes sobrescribir el objeto.
\begin{lstlisting}[style=RScript]
# Crear una nueva columna transformando una existente
mtcars |> 
	mutate(`:bn:kml` = `:bn:mpg` * `:nm:0.425`) |>    # Conversión de millas/galón a km/litro
	select(`:bn:mpg`, `:bn:kml`)                # Seleccionamos para ver el resultado
	
# Crear múltiples columnas a la vez
mtcars |> 
	mutate(
		`:bn:peso_kg`				= `:bn:wt` * 1000 * `:nm:0.453`,  # wt está en miles de libras
		`:bn:potencia_peso` = `:bn:hp` / `:bn:peso_kg`
	) |> 
	select(`:bn:wt`, `:bn:peso_kg`, `:bn:hp`, `:bn:potencia_peso`) |> 
	head(5)
\end{lstlisting}	
\end{frame}







\begin{frame}[fragile]
	También podemos usar \rscript{mutate} para usar operaciones condicionales usando \rscript{ifelse} o \rscript{case_when} que ya vimos en \quotation{estructuras de control} .
\begin{lstlisting}[style=RScript]
mtcars |> 
	mutate(`:bn:es_fuerte` = ifelse(`:bn:hp` > 100, "Potente", "Normal")) |> 
	select(`:bn:hp`, `:bn:es_fuerte`)	
\end{lstlisting}

\begin{alertblock}{}
	Utiliz\'a nombres descriptivos para tus nuevas variables. \rscript{mutate(`:bn:x` = ...)} te ahorrará tiempo al escribir pero te costará horas al leer el código meses después.
\end{alertblock}
\end{frame}
