\begin{frame}[fragile]{El verbo \texttt{filter}}
La función \rscript{filter()} nos permite quedarnos solo con las filas (observaciones) que cumplen cierta condición. Es el equivalente a usar \quotation{filtros} en Excel.

Para usar \rscript{filter()}, necesitamos expresar condiciones lógicas que sean VERDADERAS (\rscript{TRUE}) para las filas que queremos mantener.

\titleitem{Operadores de comparación}
\begin{itemize}
	\item \rscript{>}: Mayor que
	\item \rscript{<}: Menor que
	\item \rscript{>=}: Mayor o igual que
	\item \rscript{<=}: Menor o igual que
	\item \rscript{==}: Igual a
	\item \rscript{!=}: Distinto de
	\item \lstinline[style=RScript, basicstyle=\normalsize\ttfamily]|%in%|: Pertenece al conjunto (muy útil para no escribir muchos O)
\end{itemize}
\end{frame}




\begin{frame}[fragile]{Operadores lógicos y helpers}

	Además de los operadores básicos (\rscript{==}, \rscript{>}, \rscript{<}), \rscript{`:pk:dplyr`} y R base ofrecen herramientas poderosas para filtros más complejos:

	\begin{itemize}
		\item \rscript{&}: Y (ambas condiciones deben ser verdaderas). \rscript{filter(`cond1`, `cond2`)} es equivalente a \rscript{filter(`cond1` & `cond2`)}.
		\item \rscript{|}: O (al menos una condición es verdadera).
		\item \rscript{!}: No (invierte la condición).
		\item \rscript{xor(x, y)}: O exclusivo (una es verdadera, pero no ambas).
		\item \rscript{between(x, left, right)}: Un atajo para \rscript{x >= left & x <= right}.
		\item \rscript{near(x, y)}: Para comparar números con decimales (flotantes) de forma segura.
		\item \rscript{is.na(x)}: Para detectar valores perdidos (\rscript{NA}).
		\item \rscript{str_detect(columna, patron)}: (del paquete \rscript{stringr}) Para buscar texto parcial.
	\end{itemize}
\end{frame}

\begin{frame}[fragile]{Eliminar valores perdidos con \texttt{drop\_na()}}
Muchas veces queremos directamente eliminar las filas que tienen \rscript{NA} en ciertas columnas antes de operar. Si bien se puede hacer con \rscript{filter(!is.na(`:bn:columna`))}, el paquete \rscript{`:pk:tidyr`} ofrece un atajo más legible: \rscript{drop_na()}.

\begin{lstlisting}[style=RScript]
library(`:pk:tidyr`)

# Eliminar filas donde mass o height sean NA
starwars |> drop_na(`:bn:mass`, `:bn:height`)

# Sin argumentos, elimina filas con NA en CUALQUIER columna
starwars |> drop_na()
\end{lstlisting}

\begin{alertblock}{}
\rscript{drop_na()} sin argumentos puede ser muy agresivo si el dataset tiene muchas columnas con \rscript{NA}s parciales. Es preferible ser explícito sobre qué columnas importan para el análisis.
\end{alertblock}
\end{frame}




\begin{frame}[fragile]{Tidyverse \texttt{filter()} vs Base R \texttt{subsetting}}
	Es posible filtrar datos sin usar \texttt{dplyr}, utilizando la sintaxis nativa de R (corchetes \rscript{[]} o la función \rscript{subset}). Sin embargo, \rscript{filter()} suele ser preferido por su legibilidad.

	\titleitem{Comparación de sintaxis}

	Imagina que queremos las ventas de \quotation{Ana} mayores a 1000.

Opción 1: Tidyverse (\rscript{filter})
\begin{lstlisting}[style=RScript]
ventas |> filter(`:bn:vendedor` == "Ana", `:bn:monto` > 1000)
\end{lstlisting}

Ventajas: Se lee de izquierda a derecha, usa los nombres de columnas directamente.
\end{frame}



\begin{frame}[fragile]
Opción 2: R Base (Corchetes \rscript{[]})
\begin{lstlisting}[style=RScript]
ventas[ventas$vendedor == "Ana" & ventas$monto > 1000, ]
\end{lstlisting}

Desventajas
\begin{itemize}
	\item Debes repetir el nombre del dataframe (\rscript{ventas$}) cada vez.
	\item No olvides la coma al final \rscript{..., ]} (indica que quieres todas las columnas). Si la olvidas, da error o resultados inesperados.
	\item Es difícil de encadenar con otras operaciones (requiere guardar variables intermedias o anidar funciones).
\end{itemize}
\end{frame}




\begin{frame}[fragile]
Opción 3: R Base (\rscript{subset()})
\begin{lstlisting}[style=RScript]
subset(ventas, `:bn:vendedor` == "Ana" & `:bn:monto` > 1000)
\end{lstlisting}

\begin{block}{}
	\rscript{subset()} es muy similar a \rscript{filter()}, pero \rscript{filter()} es más estricto (da error si te equivocas de nombre de columna, mientras que \rscript{subset()} a veces no) y está optimizado para trabajar dentro del flujo del pipe \rscript{|>}.
\end{block}
\end{frame}




\begin{frame}[fragile]{Filtrar para visualizar}
Una de las utilidades más comunes de \rscript{filter()} es limpiar los datos antes de hacer un gráfico. Por ejemplo, si solo queremos graficar las ventas de \quotation{Ana}:
\begin{lstlisting}[style=RScript]
# Gráfico de barras solo para Ana
ventas |>
	filter(`:bn:vendedor` == "Ana") |>
	with(barplot(`:bn:monto`, names.arg = `:bn:region`, main = "Ventas de Ana por Región"))
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]{Ordenando filas con \texttt{arrange()}}
La función \rscript{arrange()} cambia el orden de las filas. Por defecto ordena de menor a mayor (ascendente). Si queremos ordenar de mayor a menor, usamos la función auxiliar \rscript{desc()}.
\begin{lstlisting}[style=RScript]
# 1. Ordenar por monto (menor a mayor)
ventas |> arrange(`:bn:monto`)
\end{lstlisting}

\begin{lstlisting}[style=RScript]
# 2. Ordenar por monto descendente (las mejores ventas primero)
ventas |> arrange(desc(`:bn:monto`))
\end{lstlisting}

\begin{lstlisting}[style=RScript]
# 3. Ordenar por vendedor y luego por monto descendente
# (Dentro de cada vendedor, sus ventas de mayor a menor)
ventas |> arrange(`:bn:vendedor`, desc(`:bn:monto`))
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]{¿Para qué sirve \texttt{arrange()}?}
Más allá de ver la tabla ordenada en la consola, \rscript{arrange()} es fundamental para:
\begin{enumerate}
	\item Rankings: Encontrar los casos \quotation{Top N} (ej. los 5 productos más vendidos).
	\item Cálculos secuenciales: Si necesitas calcular diferencias entre un mes y el anterior (\rscript{lag()} o \rscript{lead()}), el orden de las filas es crítico.
	\item Visualización ordenada: Para que un gráfico de líneas o de barras tenga sentido, a menudo necesitamos los datos en un orden específico.
\end{enumerate}

\begin{lstlisting}[style=RScript]
# Ejemplo: El Top 3 absoluto de mejores comprobantes de venta
ventas |>
	slice_max(`:bn:monto`, `:ky:n` = 3)
\end{lstlisting}

\begin{block}{}
En versiones primitivas de R se usaba \rscript{arrange()} seguido de \rscript{head()} para obtener un Top N. Hoy en día la familia \rscript{slice_} (\rscript{slice_max()}, \rscript{slice_min()}, \rscript{slice_head()}) es inmensamente superior porque \textbf{respeta los grupos} si usás \rscript{group_by()} previamente devolviendo, por ejemplo, el \quotation{Top N de cada vendedor individualmente}.
\end{block}

\begin{lstlisting}[style=RScript]
# Ejemplo Visual: Ordenando barras
# Si no ordenamos, R suele graficar en orden alfabético o de aparición
ventas |>
	arrange(desc(`:bn:monto`)) |>
	with(barplot(`:bn:monto`, names.arg = `:bn:vendedor`, main = "Ventas ordenadas por monto"))
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]{Resumiendo información con \texttt{summarise()}}
Hasta ahora, nuestras operaciones (\rscript{select()}, \rscript{mutate()}, \rscript{filter()}, \rscript{arrange()}) mantenían la estructura de los datos o simplemente reducían filas/columnas. \rscript{summarise()} (o \rscript{summarize()}) es diferente: colapsa muchas filas en una sola, calculando estadísticas de resumen.

Las funciones más comunes dentro de \rscript{summarise()} son:
\begin{itemize}
	\item \rscript{mean(`x`)}: Promedio
	\item \rscript{median(`x`)}: Mediana
	\item \rscript{sum(`x`)}: Suma total
	\item \rscript{min(`x`)} / \rscript{max(`x`)}: Mínimo y Máximo
	\item \rscript{sd(`x`)}: Desviación estándar
	\item \rscript{n()}: Cantidad de observaciones (filas)
	\item \rscript{n_distinct(`x`)}: Cantidad de valores únicos
\end{itemize}
\end{frame}



\begin{frame}[fragile]
\begin{alertblock}{Valores perdidos (\texttt{NA})}
Al calcular estadísticas como \rscript{mean()} o \rscript{sum()}, si la columna tiene al menos un \rscript{NA}, la función devolverá invariablemente \rscript{NA}. Para que las funciones omitan los datos faltantes y calculen sobre el resto, \textbf{siempre} debemos agregar el argumento \rscript{`:ky:na.rm` = TRUE} (por ejemplo, \rscript{mean(`:bn:monto`, `:ky:na.rm` = TRUE)}).
\end{alertblock}

\begin{lstlisting}[style=RScript]
# Resumen general de todas las ventas
ventas |>
	summarise(
		`:bn:total_ventas` = sum(`:bn:monto`, `:ky:na.rm` = TRUE),
		`:bn:promedio_monto` = mean(`:bn:monto`, `:ky:na.rm` = TRUE),
		`:bn:mejor_venta` = max(`:bn:monto`, `:ky:na.rm` = TRUE),
		`:bn:cantidad_transacciones` = n(),
		`:bn:vendedores_unicos` = n_distinct(`:bn:vendedor`)
	)
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
# A tibble: 1 × 5
  total_ventas promedio_monto mejor_venta cantidad_transacciones vendedores_unicos
         <dbl>          <dbl>       <dbl>                  <int>             <int>
1         8530            853        1250                     10                 3
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]{Agrupando datos: \texttt{group\_by() |> summarise()}}
\rscript{group_by()} no cambia cómo se ven los datos, pero cambia cómo actúan los verbos siguientes: las operaciones se aplican grupo a grupo de forma independiente. Combinado con \rscript{summarise()}, es el patrón más común del análisis de datos: \quotation{Agrupar por X y calcular Y}.

\begin{lstlisting}[style=RScript]
# Total de ventas y promedio de satisfacción POR VENDEDOR
ventas |>
	group_by(`:bn:vendedor`) |>
	summarise(
		`:bn:total_vendido` = sum(`:bn:monto`, `:ky:na.rm` = TRUE),
		`:bn:satisfaccion_promedio` = mean(`:bn:satisfaccion`, `:ky:na.rm` = TRUE),
		`:bn:n_ventas` = n()
	)
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
# A tibble: 3 × 4
  vendedor total_vendido satisfaccion_promedio n_ventas
  <chr>            <dbl>                 <dbl>    <int>
1 Ana               4030                  4.45        4
2 Beto              2030                  4.17        3
3 Carla             2470                  3.93        3
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]
\begin{lstlisting}[style=RScript]
# Análisis POR REGIÓN y POR PRODUCTO
ventas |>
	group_by(`:bn:region`, `:bn:producto`) |>
	summarise(
		`:bn:total_monto` = sum(`:bn:monto`, `:ky:na.rm` = TRUE),
		`:bn:total_unidades` = sum(`:bn:cantidad`, `:ky:na.rm` = TRUE)
	)
\end{lstlisting}

\begin{block}{}
Después de usar \rscript{group_by()}, el dataset queda \quotation{agrupado}. Si quieres realizar operaciones posteriores que NO deban respetar los grupos, es una buena práctica usar \rscript{ungroup()} al final de tu cadena de operaciones.
\end{block}
\end{frame}



\begin{frame}[fragile]{El atajo exploratorio definitivo: \texttt{count()}}
Contar cuántas veces aparece cada categoría es una de las operaciones más frecuentes al explorar datos. En lugar de escribir \rscript{group_by() |> summarise(n = n())}, \rscript{`:pk:dplyr`} ofrece el atajo \rscript{count()}.

\begin{lstlisting}[style=RScript]
# Cuántas ventas hizo cada vendedor
ventas |> count(`:bn:vendedor`)
\end{lstlisting}

\begin{lstlisting}[style=RScript]
# Con sort = TRUE, ordena de mayor a menor automáticamente
ventas |> count(`:bn:vendedor`, `:ky:sort` = TRUE)
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]{Operaciones por grupo: \texttt{group\_by()} |> \texttt{mutate()}}
\rscript{group_by()} no solo funciona con \rscript{summarise()}. Combinado con \rscript{mutate()}, la nueva variable se calcula dentro de cada grupo, pero el dataset conserva todas las filas originales. Es útil para calcular proporciones o participaciones relativas.

\begin{lstlisting}[style=RScript]
# Participación de cada venta dentro del total de su región
ventas |>
	group_by(`:bn:region`) |>
	mutate(
		`:bn:peso_regional` = `:bn:monto` / sum(`:bn:monto`, `:ky:na.rm` = TRUE) * 100
	) |>
	ungroup()
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]{Un ejemplo integrador}
Juntemos todo lo aprendido. Imaginemos que queremos responder: \quotation{¿Cuáles son los vendedores con mejor desempeño en ventas de productos tecnológicos (Laptops y Celulares), ignorando transacciones pequeñas (menores a 500)?}
\begin{lstlisting}[style=RScript]
ventas |>
	# 1. Filtramos: solo Laptops/Celulares y montos considerables
	filter(`:bn:producto` %in% c("Laptop", "Celular"), `:bn:monto` >= 500) |>
	# 2. Agrupamos por vendedor
	group_by(`:bn:vendedor`) |>
	# 3. Resumimos su desempeño
	summarise(
		`:bn:ingresos_totales` = sum(`:bn:monto`, `:ky:na.rm` = TRUE),
		`:bn:unidades_vendidas` = sum(`:bn:cantidad`, `:ky:na.rm` = TRUE),
		`:bn:promedio_satisfaccion` = mean(`:bn:satisfaccion`, `:ky:na.rm` = TRUE)
	) |>
	# 4. Ordenamos para ver el "ranking" (quién vendió más)
	arrange(desc(`:bn:ingresos_totales`))
\end{lstlisting}
\end{frame}
