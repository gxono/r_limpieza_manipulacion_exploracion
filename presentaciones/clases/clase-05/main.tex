\begin{frame}[fragile]{El verbo \texttt{filter}}
La funciĂ³n \rscript{filter()} nos permite quedarnos solo con las filas (observaciones) que cumplen cierta condiciĂ³n. Es el equivalente a usar \quotation{filtros} en Excel.

Para usar \rscript{filter()}, necesitamos expresar condiciones lĂ³gicas que sean VERDADERAS (\rscript{TRUE}) para las filas que queremos mantener.

\titleitem{Operadores de comparaciĂ³n}
\begin{itemize}
	\item \rscript{>}: Mayor que
	\item \rscript{<}: Menor que
	\item \rscript{>=}: Mayor o igual que
	\item \rscript{<=}: Menor o igual que
	\item \rscript{==}: Igual a
	\item \rscript{!=}: Distinto de
	\item \lstinline[style=RScript, basicstyle=\normalsize\ttfamily]|%in%|: Pertenece al conjunto (muy Ăºtil para no escribir muchos O)
\end{itemize}
\end{frame}




\begin{frame}[fragile]{Operadores lĂ³gicos y helpers}

	AdemĂ¡s de los operadores bĂ¡sicos (\rscript{==}, \rscript{>}, \rscript{<}), \rscript{`:pk:dplyr`} y R base ofrecen herramientas poderosas para filtros mĂ¡s complejos:

	\begin{itemize}
		\item \rscript{&}: Y (ambas condiciones deben ser verdaderas). \rscript{filter(`cond1`, `cond2`)} es equivalente a \rscript{filter(`cond1` & `cond2`)}.
		\item \rscript{|}: O (al menos una condiciĂ³n es verdadera).
		\item \rscript{!}: No (invierte la condiciĂ³n).
		\item \rscript{xor(x, y)}: O exclusivo (una es verdadera, pero no ambas).
		\item \rscript{between(x, left, right)}: Un atajo para \rscript{x >= left & x <= right}.
		\item \rscript{near(x, y)}: Para comparar nĂºmeros con decimales (flotantes) de forma segura.
		\item \rscript{is.na(x)}: Para detectar valores perdidos (\rscript{NA}).
		\item \rscript{str_detect(columna, patron)}: (del paquete \rscript{stringr}) Para buscar texto parcial.
	\end{itemize}
\end{frame}

\begin{frame}[fragile]{Eliminar valores perdidos con \texttt{drop\_na()}}
Muchas veces queremos directamente eliminar las filas que tienen \rscript{NA} en ciertas columnas antes de operar. Si bien se puede hacer con \rscript{filter(!is.na(`:bn:columna`))}, el paquete \rscript{`:pk:tidyr`} ofrece un atajo mĂ¡s legible: \rscript{drop_na()}.

\begin{lstlisting}[style=RScript]
library(`:pk:tidyr`)

# Eliminar filas donde mass o height sean NA
starwars |> drop_na(`:bn:mass`, `:bn:height`)

# Sin argumentos, elimina filas con NA en CUALQUIER columna
starwars |> drop_na()
\end{lstlisting}

\begin{alertblock}{}
\rscript{drop_na()} sin argumentos puede ser muy agresivo si el dataset tiene muchas columnas con \rscript{NA}s parciales. Es preferible ser explĂ­cito sobre quĂ© columnas importan para el anĂ¡lisis.
\end{alertblock}
\end{frame}




\begin{frame}[fragile]{Tidyverse \texttt{filter()} vs Base R \texttt{subsetting}}
	Es posible filtrar datos sin usar \texttt{dplyr}, utilizando la sintaxis nativa de R (corchetes \rscript{[]} o la funciĂ³n \rscript{subset}). Sin embargo, \rscript{filter()} suele ser preferido por su legibilidad.

	\titleitem{ComparaciĂ³n de sintaxis}

	Imagina que queremos las ventas de \quotation{Ana} mayores a 1000.

OpciĂ³n 1: Tidyverse (\rscript{filter})
\begin{lstlisting}[style=RScript]

\end{lstlisting}

Ventajas: Se lee de izquierda a derecha, usa los nombres de columnas directamente.
\end{frame}



\begin{frame}[fragile]
OpciĂ³n 2: R Base (Corchetes \rscript{[]})
\begin{lstlisting}[style=RScript]

\end{lstlisting}

Desventajas
\begin{itemize}
	\item Debes repetir el nombre del dataframe (\rscript{ventas$}) cada vez.
	\item No olvides la coma al final \rscript{..., ]} (indica que quieres todas las columnas). Si la olvidas, da error o resultados inesperados.
	\item Es difĂ­cil de encadenar con otras operaciones (requiere guardar variables intermedias o anidar funciones).
\end{itemize}
\end{frame}




\begin{frame}[fragile]
OpciĂ³n 3: R Base (\rscript{subset()})
\begin{lstlisting}[style=RScript]

\end{lstlisting}

\begin{block}{}
	\rscript{subset()} es muy similar a \rscript{filter()}, pero \rscript{filter()} es mĂ¡s estricto (da error si te equivocas de nombre de columna, mientras que \rscript{subset()} a veces no) y estĂ¡ optimizado para trabajar dentro del flujo del pipe \rscript{|>}.
\end{block}
\end{frame}

\begin{frame}[fragile]{Ordenando filas con \texttt{arrange()}}
La funciĂ³n \rscript{arrange()} cambia el orden de las filas. Por defecto ordena de menor a mayor (ascendente). Si queremos ordenar de mayor a menor, usamos la funciĂ³n auxiliar \rscript{desc()}.
\begin{lstlisting}[style=RScript]
# 1. Ordenar por monto (menor a mayor)
ventas |> arrange(`:bn:monto`)
\end{lstlisting}

\begin{lstlisting}[style=RScript]
# 2. Ordenar por monto descendente (las mejores ventas primero)
ventas |> arrange(desc(`:bn:monto`))
\end{lstlisting}

\begin{lstlisting}[style=RScript]
# 3. Ordenar por vendedor y luego por monto descendente
# (Dentro de cada vendedor, sus ventas de mayor a menor)
ventas |> arrange(`:bn:vendedor`, desc(`:bn:monto`))
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]{Â¿Para quĂ© sirve \texttt{arrange()}?}
MĂ¡s allĂ¡ de ver la tabla ordenada en la consola, \rscript{arrange()} es fundamental para:
\begin{enumerate}
	\item Rankings: Encontrar los casos \quotation{Top N} (ej. los 5 productos mĂ¡s vendidos).
	\item CĂ¡lculos secuenciales: Si necesitas calcular diferencias entre un mes y el anterior (\rscript{lag()} o \rscript{lead()}), el orden de las filas es crĂ­tico.
	\item VisualizaciĂ³n ordenada: Para que un grĂ¡fico de lĂ­neas o de barras tenga sentido, a menudo necesitamos los datos en un orden especĂ­fico.
\end{enumerate}

\begin{lstlisting}[style=RScript]

	slice_max(`:bn:monto`, `:ky:n` = 3)
\end{lstlisting}

\begin{block}{}
En versiones primitivas de R se usaba \rscript{arrange()} seguido de \rscript{head()} para obtener un Top N. Hoy en dĂ­a la familia \rscript{slice_} (\rscript{slice_max()}, \rscript{slice_min()}, \rscript{slice_head()}) es inmensamente superior porque \textbf{respeta los grupos} si usĂ¡s \rscript{group_by()} previamente devolviendo, por ejemplo, el \quotation{Top N de cada vendedor individualmente}.
\end{block}

\begin{frame}[fragile]{Resumiendo informaciĂ³n con \texttt{summarise()}}
Hasta ahora, nuestras operaciones (\rscript{select()}, \rscript{mutate()}, \rscript{filter()}, \rscript{arrange()}) mantenĂ­an la estructura de los datos o simplemente reducĂ­an filas/columnas. \rscript{summarise()} (o \rscript{summarize()}) es diferente: colapsa muchas filas en una sola, calculando estadĂ­sticas de resumen.

Las funciones mĂ¡s comunes dentro de \rscript{summarise()} son:
\begin{itemize}
	\item \rscript{mean(`x`)}: Promedio
	\item \rscript{median(`x`)}: Mediana
	\item \rscript{sum(`x`)}: Suma total
	\item \rscript{min(`x`)} / \rscript{max(`x`)}: MĂ­nimo y MĂ¡ximo
	\item \rscript{sd(`x`)}: DesviaciĂ³n estĂ¡ndar
	\item \rscript{n()}: Cantidad de observaciones (filas)
	\item \rscript{n_distinct(`x`)}: Cantidad de valores Ăºnicos
\end{itemize}
\end{frame}



\begin{frame}[fragile]
\begin{alertblock}{Valores perdidos (\texttt{NA})}
Al calcular estadĂ­sticas como \rscript{mean()} o \rscript{sum()}, si la columna tiene al menos un \rscript{NA}, la funciĂ³n devolverĂ¡ invariablemente \rscript{NA}. Para que las funciones omitan los datos faltantes y calculen sobre el resto, \textbf{siempre} debemos agregar el argumento \rscript{`:ky:na.rm` = TRUE} (por ejemplo, \rscript{mean(`:bn:monto`, `:ky:na.rm` = TRUE)}).
\end{alertblock}

		`:bn:mejor_venta` = max(`:bn:monto`, `:ky:na.rm` = TRUE),
		`:bn:cantidad_transacciones` = n(),
		`:bn:vendedores_unicos` = n_distinct(`:bn:vendedor`)
	)
\end{lstlisting}

\begin{frame}[fragile]{Operaciones por grupo: \texttt{group\_by()} |> \texttt{mutate()}}
\rscript{group_by()} no solo funciona con \rscript{summarise()}. Combinado con \rscript{mutate()}, la nueva variable se calcula dentro de cada grupo, pero el dataset conserva todas las filas originales. Es Ăºtil para calcular proporciones o participaciones relativas.
