	\begin{frame}[fragile]{Factores (Factor)}
		Los factores son un tipo de dato especial para variables categóricas (como \quotation{Nivel educativo}, \quotation{Sexo}, \quotation{Provincia}). Aunque parecen texto, internamente R los guarda como números enteros asociados a etiquetas.
		
		\titleitem{¿Por qué usarlos?}
		Son útiles para modelos estadísticos y para ordenar gráficos de forma no alfabética (por ejemplo, \quotation{Bajo}, \quotation{Medio}, \quotation{Alto}). También son más eficientes a la hora de guardar grandes bases de datos.
	\end{frame}

	\begin{frame}[fragile]{Estructuras de datos compuestas}
Hasta ahora hemos trabajado con vectores y matrices, que tienen una restricción fuerte: todos sus elementos deben ser del mismo tipo (todos lógicos, todos números o todos texto). Sin embargo, en la vida real, los conjuntos de datos suelen venir mezclados. Por ejemplo, en una encuesta se puede preguntar por el sexo (categórico/factor), por la edad (numérico), o por si tiene trabajo formal (lógico).

Para manejar esto, R nos ofrece las listas y los data frames.
	\end{frame}
	
	
	
	
	
	\begin{frame}[fragile]{Listas}
Las listas son objetos muy flexibles (y esto no necesariamente es una ventaja) que pueden contener cualquier cosa: vectores, matrices, otras listas, funciones, etc. No requieren que sus elementos sean del mismo tipo \textbf{ni de la misma longitud}.

\begin{lstlisting}[style=RScript]
mi_lista <- list(
	nombre = "Lucía",
	edad = 30,
	notas = c(8, 9, 10),
	aprobado = TRUE
)

mi_lista
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
$nombre
[1] "Lucía"

$edad
[1] 30

$notas
[1]  8  9 10

$aprobado
[1] TRUE
\end{lstlisting}
	\end{frame}
	
	
	
	
\begin{frame}[fragile]{Acceso a elementos}
	A diferencia de los vectores, las listas tienen dos tipos de corchetes:

\begin{itemize}
	\item \rscript{[]} (corchete simple): Devuelve una sub-lista. \textbf{Importante!} El resultado sigue siendo una lista.
	\item \rscript{[[]]} (doble corchete): Extrae el contenido real del elemento.
	\item \rscript{$} (signo peso): Accede a los elementos por nombre (si lo tienen). Extrae como \rscript{[[]]}.
\end{itemize}	


\begin{lstlisting}[style=RScript]
mi_lista[1]       # Devuelve una lista con el elemento "nombre"
typeof(mi_lista[1])
\end{lstlisting}

\end{lstlisting}

\end{lstlisting}

\end{lstlisting}

\begin{frame}[fragile]{Data frames}
	El data frame es, sin lugar a dudas, \textbf{la estructura más importante} para el análisis de datos en R. Si venís del mundo de Excel, podés pensarlo como una hoja de cálculo, donde cada fila es una observación y cada columna es una variable.
	
	Es la estructura fundamental porque agrupa vectores de diferentes tipos (caracteres, números, lógicos) en un mismo objeto rectangular, manteniendo la integridad de los datos:
	\begin{enumerate}
		\item Rectangularidad: Todas las columnas deben tener la misma longitud.
		\item Heterogeneidad de columnas: La columna \quotation{Edad}  puede ser numérica mientras que \quotation{Nombre}  es texto.
		\item Homogeneidad interna: Dentro de una misma columna, todos los datos deben ser del mismo tipo.		
	\end{enumerate}
		
	Técnicamente, en R un data frame no es más que una lista de vectores de igual longitud, que se visualiza como una tabla. Esta dualidad (lista/tabla) es lo que lo hace tan potente.
\end{frame}

\begin{frame}[fragile]{Estructuras de control de flujo}
	El control de flujo nos permite decidir qué código ejecutar y cuántas veces. Sin embargo, es importante hacer una distinción crucial en R:

	\begin{alertblock}{}
		A diferencia de lenguajes como Julia o Python, donde los bucles (\rscript{for}, \rscript{while}) son el pan de cada día, en R intentamos evitarlos siempre que sea posible.
	\end{alertblock}
	
	La filosofía de R es vectorial: en lugar de iterar elemento por elemento, aplicamos operaciones a todo el vector simultáneamente. Esto no solo hace el código más legible, sino mucho más eficiente. Aun así, estas estructuras existen y son fundamentales para cuando la vectorización no es posible o para tareas de programación más general (como por ejemplo, implementar simulaciones complejas).
\end{frame}






\begin{frame}[fragile]{if y else}
	Permiten ejecutar un bloque de código solo si se cumple una condición lógica (\rscript{TRUE}). 
	\begin{alertblock}{}
		\rscript{if} solo evalúa un único valor lógico, no un vector (para vectores usaremos \rscript{ifelse}).
	\end{alertblock}
\begin{lstlisting}[style=RScript]
if (condicion) {
	# Instrucción si es VERDADERO
} else {
	# Instrucción si es FALSO
}	
\end{lstlisting}
	
\begin{lstlisting}[style=RScript]
promedio <- 5.5

if (promedio >= 6) {
	print("Aprobado")
} else {
	print("Desaprobado")
}	
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] "Desaprobado"	
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]
Se pueden encadenar condiciones con \rscript{else if}:
\begin{lstlisting}[style=RScript]
nota <- 8

if (nota == 10) {
	mensaje <- "Excelente"
} else if (nota >= 8) {
	mensaje <- "Muy bueno"
} else if (nota >= 6) {
	mensaje <- "Bueno"

print(mensaje)
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] "Muy bueno"
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]{Condiciones vectorizadas}
Como \rscript{if} no funciona bien con vectores largos, R tiene una función optimizada llamada \rscript{ifelse()}. Esta función toma un vector de condiciones y devuelve un vector de resultados del mismo largo.

\begin{lstlisting}[style=RScript]
ifelse(test, si_verdadero, si_falso)
\end{lstlisting}

\begin{lstlisting}[style=RScript]
notas_curso <- c(4, 9, 6, 2, 10)
estado <- ifelse(notas_curso >= 6, "Aprobado", "Desaprobado")
estado
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] "Desaprobado" "Aprobado"    "Aprobado"    "Desaprobado" "Aprobado"   
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]
	Esto es equivalente a escribir un bucle \rscript{for} que recorra cada nota, pero es mucho más rápido y se escribe en una sola línea. Aunque aun no hemos visto el bucle \rscript{for}, adjunto un ejemplo de cómo debería implementarse lo mismo utilizando esta aproximación, solo para comparar las extensiones.
\begin{lstlisting}[style=RScript]
notas_curso <- c(4, 9, 6, 2, 10)
estado <- logical(length(notas_curso))
for (indice in 1:length(notas_curso)){
	estado[indice] <- notas_curso[indice] >= 6
}

estado	
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] FALSE  TRUE  TRUE FALSE  TRUE	
\end{lstlisting}

\begin{frame}[fragile]{Bucles for}
Los bucles \rscript{for} permiten repetir un bloque de código una cantidad determinada de veces. Se suele iterar sobre una secuencia numérica o sobre los elementos de un vector.

\begin{lstlisting}[style=RScript]
for (`variable` in secuencia) {
	# Código a repetir
}	
\end{lstlisting}

\begin{lstlisting}[style=RScript]
animales <- c("Gato", "Perro", "Pájaro")

for (animal in animales) {
	print(paste("El", animal, "es una mascota"))
}	
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] "El Gato es una mascota"
[1] "El Perro es una mascota"
[1] "El Pájaro es una mascota"
\end{lstlisting}
\end{frame}





\begin{frame}[fragile]
\begin{lstlisting}[style=RScript]
valores <- c(2, 4, 6)
cuadrados <- numeric(length(valores)) # Pre-asignar espacio es buena práctica

for (i in 1:length(valores)) {
	cuadrados[i] <- valores[i]^2
}
cuadrados
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
	[1]  4 16 36
\end{lstlisting}

	¿Qué hace \rscript{numeric()}? Comparelo con \rscript{logical()}.
\end{block}
\end{frame}



\begin{frame}[fragile]
	En R, siempre intenta usar operaciones vectorizadas antes de escribir un bucle \rscript{for}. Los bucles son útiles para tareas recursivas (donde el paso actual depende del anterior) o procesos complejos que no son puramente matemáticos (como leer múltiples archivos), pero para cálculos directos son menos eficientes y más verbosos.
	
	En este caso, es más eficiente hacer
\begin{lstlisting}[style=RScript]
valores <- c(2, 4, 6)

cuadrados <- valores^2

cuadrados
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1]  4 16 36
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]
	Como dije antes, a menos que est\'es construyendo una simulaci\'on compleja, no va a ser necesario que recurras a cosas como un \rscript{for}.
\end{frame}
