	\begin{frame}{Instalación y configuración del entorno (R y RStudio)}
		Para trabajar con R, necesitamos instalar tres programas:
		\begin{enumerate}

			\url{https://cran.r-project.org/bin/windows/Rtools/}
		\end{enumerate}
	\end{frame}

	\begin{frame}[fragile]{Introducción al entorno RStudio}
		Al abrir RStudio, verás la pantalla dividida en paneles:
		\begin{enumerate}
			\item Consola (Console): Es donde R ejecuta las instrucciones y muestra los resultados inmediatos. Si escribes \rscript{2 + 2} y das \verb|Enter|, verás \rscript{4}.
			
			\item Editor de Código (Source/Script): Aquí escribimos nuestros programas (scripts). Es como un bloc de notas inteligente. El código aquí no se ejecuta hasta que tú lo ordenas (usualmente con \verb|Ctrl + Enter| para ejecutar un chunk).

			\item Archivos/Gráficos/Ayuda (Files/Plots/Help): Un panel multifunción para navegar carpetas, ver visualizaciones y consultar documentación.

	\begin{frame}[fragile]{Buenas prácticas: Organización del Script}
		Un \textit{script} es un fichero de texto en el que organizaremos el código. En ocasiones, un proyecto puede tener varios scripts involucrados.
		
		Para crear un nuevo script iremos a \verb|File > New file > R Script|.
		
		Un \textit{script} ordenado es un \textit{script} feliz (y reproducible). Segu\'i estas reglas de oro:
		\begin{enumerate}
			\item Encabezado: Comienza siempre cargando todos los paquetes necesarios (\rscript{library(...)}) en las primeras líneas.
			\item Comentarios: Usa \rscript{#} para explicar el \textit{por qué} de tu código, no el \textit{qué} (el código ya dice qué hace).
		\end{enumerate}
		
		\begin{lstlisting}[style=RScript]
# install.packages("dplyr")
library(dplyr) # Para poder usar lag.

datos <- c(150, 100, 50, 15, 150)

#   Variación relativa de un vector <- Esto dice "qué". Evitar a menos
# que sea una definición extensa y requiera explicación detallada en
# cada paso.
variacion_relativa = function(x) {
	return ((x - lag(x)) / lag(x))
}

`:fn:variacion_relativa`(datos)
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1]         NA -0.3333333 -0.5000000 -0.7000000  9.0000000
\end{lstlisting}
	\end{frame}

\begin{frame}[fragile]{Nombre de los objetos}
	R es muy flexible, pero seguir convenciones hace que tu código sea legible para tu \guillemetleft yo\guillemetright\ del futuro (\shaky{y para tu profe que te va a tener que corregir} \emoji{upside-down-face}).
	
	\titleitem{Reglas duras (condiciones necesarias)}
	\begin{itemize}
		\item Deben comenzar con una letra.
		\item No pueden contener espacios (usa guion bajo \rscript{_}).
		\item R distingue mayúsculas de minúsculas (\rscript{Dato} es distinto de \rscript{dato}).
	\end{itemize}
	
	\titleitem{Convenciones de estilo (condiciones recomendadas)}
	\begin{itemize}
		\item snake\_case (recomendado por tidyverse): \rscript{mi_variable}, \rscript{promedio_edad}. Es fácil de leer.
		\item camelCase: \rscript{miVariable}, \rscript{promedioEdad}. Común en \textbf{otros} lenguajes.
		\item dot.case: \rscript{`mi.variable`}. Histórico en R, pero a veces confunde con métodos de objetos o con variables ocultas.
	\end{itemize}
\end{frame}

\begin{frame}[fragile]{Vectores y tipos de datos (numéricos, caracteres, lógicos)}
	Comprender los tipos de datos es fundamental cuando trabajamos en R, especialmente porque más adelante estaremos procesando data frames (marco de datos) que contienen múltiples variables. Cada variable en un data frame tiene un tipo específico (números, texto, categorías, etc.), y conocer estos tipos nos permitirá aplicar las operaciones correctas y evitar errores al procesarlos.
	
	Del mismo modo, es esencial entender los vectores, ya que la información en R está alojada en esta estructura: cada columna de un dataframe es un vector, y las operaciones que apliquemos sobre ellos determinarán el análisis que podamos realizar.
\end{frame}

\begin{frame}[fragile]
	Aunque algunos posibles problemas se solucionan por coerción. La coerción ocurre cuando un tipo de dato se convierte en otro para complacer a una función. La conversión ocurre en este orden:
	\begin{center}
		\rscript{logical -> integer -> numeric -> character}
	\end{center}
	Por ejemplo, supongamos que tenemos \rscript{`b1` <- TRUE}, \rscript{`b2` <- TRUE} y \rscript{`b3` <- FALSE}. La operación suma (bajo el operador \rscript{+}) no está definida para datos lógicos. Aun así, observe el siguiente bloque
\begin{lstlisting}[style=RScript]
`b1` <- TRUE; `b2` <- TRUE; `b3` <- FALSE

`b1` + `b2` + `b3`
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] 2
\end{lstlisting}
\end{frame}




\begin{frame}[fragile]
	Lo que ocurre es que la coerción convierte los datos en enteros (\rscript{integer}) y aplica la suma. Por defecto, \rscript{TRUE} se convierte en \rscript{1} y \rscript{FALSE} se convierte en \rscript{0}.
\end{frame}

\end{frame}

\begin{frame}[fragile]{Aritmética vectorizada: Operaciones elementales sin bucles}
	Esta es una de las características más potentes de R y lo diferencia de lenguajes como C o Python clásico. R está diseñado para operar sobre vectores completos de una sola vez (similar al concepto SIMD - Single Instruction, Multiple Data).
	
	\titleitem{¿Por qué es importante?}
	\begin{enumerate}
		\item Claridad: El código se lee como una operación matemática (\guillemetleft precio por cantidad\guillemetright ), no como una instrucción informática (\guillemetleft toma el elemento i, multiplícalo, guárdalo, repite\guillemetright ).
		\item Eficiencia: Las operaciones vectorizadas están optimizadas en C, siendo mucho más rápidas que un bucle explícito escrito en R.
	\end{enumerate}
	
\begin{lstlisting}[style=RScript]
# R
precios <- c(100, 200, 300)
precios_con_iva <- precios * `:nm:1.21`
\end{lstlisting}

\begin{lstlisting}[style=JuliaScript]
# Julia
precios = [100, 200, 300]
precios_con_iva = precios .* 1.21
\end{lstlisting}

\begin{lstlisting}[style=PythonScript]
# Python
precios = [100, 200, 300]
precios_con_iva = [p * 1.21 for p in precios]
\end{lstlisting}
\end{frame}

\end{frame}





\begin{frame}[fragile]{Matrices}
	Las matrices son extensiones de los vectores a dos dimensiones (filas y columnas). Al igual que los vectores, todos sus elementos deben ser del mismo tipo (todos números, todos caracteres, etc.).
\end{frame}


\begin{frame}[fragile]{Creación de matrices}
Usamos la función \rscript{matrix()}, especificando los datos y las dimensiones.
\begin{lstlisting}[style=RScript]
# Crear una matriz de 3 filas y 2 columnas con números del 1 al 6
mi_matriz <- matrix(`:ky:data` = 1:6, `:ky:nrow` = 3, `:ky:ncol` = 2)
mi_matriz
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
     [,1] [,2]
[1,]    1    4
[2,]    2    5
[3,]    3    6	
\end{lstlisting}
\begin{lstlisting}[style=RScript]
# Por defecto llena por columnas. Para llenar por filas:
matriz_filas <- matrix(1:6, `:ky:nrow` = 3, byrow = TRUE)	
\end{lstlisting}
\end{frame}





\begin{frame}[fragile]{Propiedades y acceso}
	Podemos consultar sus dimensiones y acceder a elementos específicos usando corchetes \rscript{[fila, columna]}.
\begin{lstlisting}[style=RScript]
dim(mi_matriz)      # Devuelve c(3, 2) -> cantidad de filas y de columnas
nrow(mi_matriz)     # Número de filas
ncol(mi_matriz)     # Número de columnas

# Indexación (Slicing)
mi_matriz[1, 2]     # Elemento en fila 1, columna 2
mi_matriz[2,  ]     # Toda la fila 2
mi_matriz[ , 1]     # Toda la columna 1

# O varias selecciones a la vez
mi_matriz[ , c(1, 5)]     # Error, porque la columna 5 no existe
\end{lstlisting}
\end{frame}

t(mi_matriz)        # Transpuesta (cambia filas por columnas)

colMeans(mi_matriz) # Promedio de cada columna	
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]{Acceso a elementos de un vector (indexación)}
	Una vez que tenemos datos en un vector, a menudo necesitamos seleccionar partes específicas de él. Para esto usamos los corchetes \rscript{[]}.
\end{frame}


\begin{frame}[fragile]{Indexación por posición (enteros positivos)}
	Indicamos la posición del elemento que queremos recuperar.
\begin{lstlisting}[style=RScript]
letras <- c("A", "B", "C", "D", "E")

letras[1]       # El primer elemento ("A")
letras[3]       # El tercer elemento ("C")

# Podemos pedir varios a la vez pasando un vector de posiciones
letras[c(1, 5)] # El primero y el quinto ("A", "E")
\end{lstlisting}
\end{frame}


\begin{frame}[fragile]{Indexación por exclusión (enteros negativos)}
	Si usamos números negativos, R devuelve \textbf{todo menos} lo que indicamos.
\begin{lstlisting}[style=RScript]
letras <- c("A", "B", "C", "D", "E")

letras[-1]       # Todo menos el primero
letras[-c(4, 5)] # Todo menos el cuarto y el quinto
\end{lstlisting}
\end{frame}



\begin{frame}[fragile]{Indexación lógica (filtrado)}
	Esta es quizás la forma más potente. Usamos un vector de \rscript{TRUE} y \rscript{FALSE} (usualmente resultado de una condición) para quedarnos solo con los elementos que cumplen la condición (\rscript{TRUE}).
\begin{lstlisting}[style=RScript]
valores <- c(10, 5, 20, 2)

# Paso 1: Crear la condición (vector lógico)
condicion <- valores > 5
condicion 
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] TRUE FALSE TRUE FALSE	
\end{lstlisting}
\begin{lstlisting}[style=RScript]
# Paso 2: Usar la condición para filtrar
valores[condicion]
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] 10 20
\end{lstlisting}
\begin{lstlisting}[style=RScript]
# Todo en una sola línea (lo más común):
valores[valores > 5]
\end{lstlisting}
\begin{lstlisting}[style=RConsole]
[1] 10 20	
\end{lstlisting}
\end{frame}

\begin{frame}[fragile]{Ejercicios}
	Intenta resolverlos paso a paso en un \'unico script.
	
	\begin{enumerate}
		\item Crea un objeto con tu año de nacimiento y otro con el año actual (como número). Calcula tu edad aproximada restando ambos objetos y guárdala en una nueva variable.
		\item Crea un vector con las temperaturas máximas de la semana: 24, 28, 31, 25, 22. Convierte todo el vector a grados Fahrenheit ($F = C \times 1.8 + 32$) en una sola línea.
