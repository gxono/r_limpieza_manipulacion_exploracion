	\begin{frame}[fragile]{\texttt{scale\_x\_continuous()} y \texttt{scale\_y\_continuous()}}
		Controlan los ejes continuos: dĂ³nde aparecen las marcas, cĂ³mo se formatean las etiquetas y cuĂ¡les son los lĂ­mites visibles.

    limits  = c(`:nm:0`, `:nm:70000`)

    `:ky:labels` = scales::label_number(suffix = " aĂ±os")

\end{lstlisting}

		\begin{block}{}
			El paquete \rscript{`:pk:scales`} provee funciones de formato listas para usar: \rscript{scales::comma} (separador de miles), \rscript{scales::percent} (porcentaje), \rscript{scales::dollar} (moneda), \rscript{scales::label\_number()} (formato personalizado).
		\end{block}
	\end{frame}


	\begin{frame}[fragile]{\texttt{scale\_x\_date()}: ejes de fechas}

	\begin{frame}[fragile]{\texttt{scale\_color\_*()}: paletas de color}
		Hay tres tipos de escala de color segĂºn el tipo de variable: \textbf{discreta} (categorĂ­as), \textbf{continua} (valores numĂ©ricos) y \textbf{divergente} (valores con un punto medio significativo).

		\begin{exampleblock}{}

	\begin{frame}[fragile]{\texttt{theme()}: personalizaciĂ³n del tema}
		\rscript{theme()} modifica cualquier elemento no relacionado con los datos: tipografĂ­a, grillas, leyenda, fondo, mĂ¡rgenes. Se usa \textbf{despuĂ©s} del tema base.

\begin{lstlisting}[style=RScript]

                           `:ky:fill` = `:bn:continent`)) +
  geom_boxplot(alpha = `:nm:0.7`) +
  labs(`:ky:title` = "PBI per cĂ¡pita por continente (2007)",
       x = NULL, y = "PBI per cĂ¡pita (USD)") +
  theme_minimal() +                      # primero el tema base
  theme(                                 # luego las modificaciones
    plot.title         = element_text(size = `:nm:13`, face = "bold"),
    axis.text.x        = element_text(size = `:nm:10`),
    legend.position    = "none",
    panel.grid.major.x = element_blank()
  )
\end{lstlisting}

		\begin{alertblock}{}
			\rscript{`:fn:theme_*`()} debe ir \textbf{antes} de \rscript{theme()}. Si se invierte el orden, el tema base sobreescribe las modificaciones y estas no tienen efecto.
		\end{alertblock}
	\end{frame}

	\begin{frame}[fragile]{\texttt{geom\_text()} y \texttt{geom\_label()}}
		A diferencia de \rscript{annotate()}, \rscript{geom\_text()} y \rscript{geom\_label()} leen del dataset: etiquetan cada observaciĂ³n con el valor de una variable mapeada a \rscript{`:ky:label`} dentro de \rscript{aes()}.

  filter(`:bn:lifeExp` < `:nm:50` | `:bn:gdpPercap` > `:nm:35000`)

  geom_point(aes(`:ky:color` = `:bn:continent`), alpha = `:nm:0.6`) +
  geom_label(
    data = extremos,
    aes(label = `:bn:country`),
    size = `:nm:2.5`, alpha = `:nm:0.8`
  ) +
  scale_x_log10()
\end{lstlisting}

		\begin{exampleblock}{}
			Etiquetar todos los puntos produce solapamiento. La estrategia habitual es pasar un subconjunto con \rscript{data = extremos}. El paquete \rscript{`:pk:ggrepel`} (\rscript{geom\_text\_repel()}) ajusta automĂ¡ticamente las posiciones para evitar superposiciones: es la soluciĂ³n estĂ¡ndar cuando hay muchas etiquetas.
		\end{exampleblock}
	\end{frame}
